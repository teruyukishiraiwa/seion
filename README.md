# Seion（静音）

- アプリ: https://seion-writing.pages.dev
- GitHub: https://github.com/teruyukishiraiwa/seion
- Version: 1.0.0

詩文・散文・SNS投稿用の文章を、静謐な画面で執筆するためのライティングアプリケーション。
正式名称: Seion（静音）。旧称: Quiet Manuscript。

クライアントサイド完結。データは localStorage に保存されます。サーバー・ログイン・クラウド同期は不要です。

## 技術スタック

- React 18 + TypeScript
- Vite 5
- Tailwind CSS 3（`darkMode: "class"`）
- html-to-image（SNS画像のPNG出力）

## セットアップ

```sh
npm install
npm run dev      # 開発サーバー (http://localhost:5173)
npm run build    # 型チェック + 本番ビルド (dist/)
npm run preview  # ビルド結果のプレビュー
```

## 主な機能（MVP）

- ノートの作成 / 選択 / 論理削除 / 復元 / 完全削除、検索、リスト / カード表示
- タイトル・本文の編集（リッチテキスト対応の contenteditable）
- 本文の一部分への装飾（選択するとフローティングツールバーが出現）
  - 太字 / 斜体 / 下線 / 打消し線
  - 文字色（プリセット＋カスタムカラーピッカー）
  - 蛍光色（マーカー）
  - 装飾の解除
  - 装飾はエディタ・SNS出力・ページ分割すべてに反映
- localStorage への自動保存（デバウンス）
- フォント切り替え（明朝系 / ゴシック系 / Serif / Sans、OS標準フォント優先）
- 文字サイズ・行間・文字間・本文幅の調整
- ライト / ダークモード
- 集中モード（左右パネルを隠す）
- 投稿比率表示（1:1 / 9:16 / 16:9 の外形比率でライティング領域を表示）
- 背景画像のアップロード / 削除、オーバーレイ（白・黒）と濃度の調整
- SNS画像出力（1:1 = 1080×1080 / 9:16 = 1080×1920 / 16:9 = 1920×1080）
  - 現在のノートからプレビューを生成し、PNG としてエクスポート
  - 本文の左寄せ / 中央を切り替え可能
  - 長い本文は署名（`Seion`）に重ならないよう上端から流し込み、署名の上で自然にクリップ
  - **複数枚に分割**: 1枚に収まらない文章を自動でページ分割し、複数のPNG
    （`タイトル_9x16_1of3.png` など）として連続出力。タイトルは1枚目のみ、
    各ページ右下に `1 / 3` のページ番号を表示。インスペクタでプレビューをページ送り可能

## 実装メモ

- 画像出力は、SNSカードを実寸（例 1080×1920）で画面外に描画し、`html-to-image` の
  `toSvg` で SVG 化したのち、`<img>` の `onload` を待って自前で canvas に描画して PNG 化しています。
  `toPng` が内部で使う `img.decode()` は一部ブラウザで大きな data-URI SVG に対して停止することが
  あるため、それを避けています。OS標準フォントのみ使うため `skipFonts: true` を指定しています。
- リッチテキストは、本文を「1行=1 `<div>`、インラインに装飾タグ」という正規化ブロックHTMLで保持し、
  エディタ表示・SNS描画・ページ分割すべてが同じ表現を共有します。装飾は `document.execCommand`
  （`styleWithCSS` 有効）で適用し、依存ライブラリを増やしていません。`Note.body`（プレーン）と
  `Note.bodyHtml`（リッチ）を同期し、`bodyHtml` 未設定の旧ノートは `body` から自動変換します。
- ページ分割は各行ブロックの高さを1度ずつ計測し、**最小ページ数**を保ったまま二分探索で
  ページごとの高さ上限を均等化します。これにより最終ページに1行だけ残る「孤立行」を避けます。


## データ保全と操作

- 保存済みの空配列は「ノートなし」としてそのまま復元し、初期サンプルを再生成しません。
- ノートと設定の保存失敗は画面下部の通知で知らせます。手動保存は待機中の自動保存と競合しません。
- 削除は `trashedAt` を設定する論理削除です。ゴミ箱から復元・完全削除・一括削除できます。
- 左下のフォルダ / ゴミ箱で表示対象を切り替え、Grid アイコンでリスト / カード表示を切り替えます。
- 右上メニューは新規作成、保存、エクスポート、集中モード、テーマ、クレジットをまとめ、矢印キー・Home / End・Escape・外側クリックに対応します。

### 署名画像

- PNG / WebP の署名画像を設定し、本文末尾の右端へ表示できます。
- 表示切替、画像幅（本文領域の8〜50%、初期値20%）、本文との間隔（1〜15%、初期値4%）をインスペクタから調整できます。
- 複数ページ出力では署名は最終ページだけに表示され、ページ分割時に署名分の高さを確保します。
- 既存のカード下部中央の `Seion` 表記はアプリのフッターとして維持されます。


### 背景のぼかし

背景画像には0〜40のぼかしを設定できます。強度に応じて画像を自動拡大し、
ぼかしによるカード端の隙間を防止します。エディタ、SNSプレビュー、
PNG出力で同じ相対強度を使用し、白・黒オーバーレイとは独立して調整できます。


### 投稿比率表示

ステータスバーの「投稿比率で表示」を有効にすると、選択中のSNS出力比率に合わせてライティングキャンバスを中央へ配置します。左右ペイン、集中モード、ウィンドウサイズ、比率設定の変更へ追従し、背景・ぼかし・オーバーレイ・署名はキャンバス内に限定されます。これは外形比率を一致させる機能であり、文字組み・改ページまでの完全なWYSIWYGではありません。既定値はOFFです。

WYSIWYG化の第一段階として、投稿比率表示はSNSカードと同じ論理解像度と cardGeometry を使用します。タイトル・本文・背景・ぼかし・署名・フッターは出力座標上で描画してから画面内へ縮小されるため、単一ページでは出力と同じ文字組みになります。

投稿比率表示では、複数枚分割が有効な場合に現在の総ページ数と1ページ目の境界ガイドを表示します。日本語IMEの変換中は本文保存とページ再計算を保留し、変換確定時に一度だけ反映します。編集DOMはカーソル安定性のため単一のまま維持します。

投稿比率表示の表示倍率は、自動・50%・75%・100%から選択できます。固定倍率ではキャンバスを縦横スクロールでき、Ctrl/Cmd＋ホイールで倍率を段階変更できます。表示倍率は編集画面だけの状態で、PNG出力や保存設定には影響しません。

投稿比率モード切替時は、新しい編集DOMへ本文HTMLを再適用します。EditorとInspectorは共通のfitCardScaleで表示倍率を計算し、外枠へ明示的なaspect-ratioを設定しています。複数枚分割時は1ページ目を編集カード、2ページ目以降を同倍率の出力カードとして縦に表示し、署名は最終ページだけに表示します。

1:1・16:9の分割表示差を修正しました。編集カードは全本文DOMを保持したまま、pages[0]より後のブロックを表示上だけ隠します。これにより1ページ目もプレビューと同じ改ページ境界になり、非表示部分の本文データは失われません。ページ番号の書式もSnsCardと統一しています。

### PWAインストール

- Chromium系ではブラウザのインストール可能判定後、編集開始時または再訪時に控えめな案内を表示します。
- 「今はしない」を選ぶと7日間自動表示しません。右上メニューから再度開けます。
- iOS Safariでは共有メニューから「ホーム画面に追加」する手順を表示します。
- 本番ビルドではService Workerを登録し、取得済みアプリ資産をキャッシュします。編集中の強制リロードは行いません。
- 開発サーバーではキャッシュ競合を避けるためService Workerを登録しません。
